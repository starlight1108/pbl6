# 接口测试说明

## 测试框架

- 测试框架：**pytest**
- 测试覆盖：用户注册、用户登录、商品发布、商品浏览、商品详情

---

## 测试文件结构

```
backend/
├── pytest.ini          # pytest配置文件
└── tests/
    ├── __init__.py
    ├── conftest.py   # 测试fixture
    ├── test_auth.py    # 认证接口测试
    └── test_product.py # 商品接口测试
```

## 运行方式

### 安装依赖
```bash
cd backend
pip install -r requirements.txt
```

### 执行所有测试
```bash
pytest
```

### 执行指定测试
```bash
# 只测认证接口
pytest tests/test_auth.py

# 只测商品接口
pytest tests/test_product.py

# 显示详细输出
pytest -v
```

### 查看测试覆盖率
```bash
pytest -v --tb=long
```

## 测试Fixture (`conftest.py`)

| Fixture | 说明 |
|---------|------|
| `app` | 创建测试应用，使用内存数据库 |
| `client` | HTTP测试客户端 |
| `auth_headers` | 自动注册登录，返回带Token的请求头 |

---

## 测试特点

1. ✅ **隔离测试** - 每个测试用例之间数据隔离，互不影响
2. ✅ **内存数据库** - 使用 SQLite 内存数据库，测试后自动清理
3. ✅ **自动登录** - `auth_headers` fixture 自动完成注册登录
4. ✅ **边界测试** - 正常流程 + 异常场景全覆盖
