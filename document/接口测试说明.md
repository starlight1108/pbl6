# 接口测试说明

## 测试框架

- 测试框架：**pytest 7.4.4**
- 测试覆盖：**26个测试用例** — 用户注册、用户登录、商品发布、商品浏览、商品详情、用户头像


## 运行方式

### 安装依赖
```bash
cd backend
pip install -r requirements.txt
```

### 执行所有测试
```bash
pytest
```

### 执行指定模块
```bash
# 仅认证接口（8个用例）
pytest tests/test_auth.py -v

# 仅商品接口（12个用例）
pytest tests/test_product.py -v

# 仅用户接口（6个用例）
pytest tests/test_user.py -v
```

### 常用选项
```bash
pytest -v                  # 详细输出
pytest --tb=short          # 简短错误回溯
pytest -k "keyword"        # 按关键词筛选用例
pytest -x                  # 首个失败即停止
```

## 测试 Fixture — `conftest.py`

### 应用 Fixture

| Fixture | 作用域 | 说明 |
|---------|-------|------|
| `app` | function | 创建测试Flask应用，SQLite内存数据库，**每个用例独立重建** |
| `client` | function | Flask HTTP测试客户端 |

### 认证 Fixture

| Fixture | 作用域 | 说明 |
|---------|-------|------|
| `auth_headers` | function | 自动注册+登录，返回 `{"Authorization": "Bearer <token>"}` 请求头 |

### 数据 Fixture

| Fixture | 作用域 | 说明 |
|---------|-------|------|
| `create_test_products` | function | 插入3条测试商品（书籍、数码、运动，不同分类） |
| `create_many_products` | function | 插入15条商品用于分页测试 |

### 自动清理机制

```
测试结束后自动清理:
  1. SQLite内存数据库自动销毁
  2. AVATAR_FOLDER 和 PRODUCT_FOLDER 中的测试临时文件
```

## 测试特点

| 特性 | 说明 |
|------|------|
| ✅ **完全隔离** | 每个用例独立数据库，互不干扰 |
| ✅ **内存数据库** | 使用 SQLite `:memory:`，测试后自动销毁 |
| ✅ **自动认证** | `auth_headers` fixture 自动完成注册/登录流程 |
| ✅ **边界覆盖** | 正常流程 + 异常场景（缺字段、重复、不存在、未认证）全覆盖 |
| ✅ **文件上传测试** | 使用 Pillow 动态生成图片模拟上传，测试后自动删除 |
| ✅ **分页测试** | 动态插入15条数据验证分页逻辑 |
| ✅ **默认值测试** | 验证默认头像、默认商品图、默认分类等降级策略 |
